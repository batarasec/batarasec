# Security Policy

## Reporting a vulnerability

Please do not report security vulnerabilities through public GitHub issues.

For now, report suspected vulnerabilities privately through the official BataraSec security contact that will be published before the public Community repository is opened.

When reporting, include:

- Affected component or version.
- Reproduction steps.
- Impact assessment.
- Relevant logs, screenshots, or proof-of-concept details.
- Whether the issue affects Community, Enterprise, or both if known.

## Scope

Security reports may cover:

- BataraSec Community installer and Docker Compose deployment.
- BataraSec API, web, worker, and agent behavior distributed through Community images.
- Authentication, authorization, tenant isolation, data exposure, and supply-chain issues.

Out of scope:

- Denial-of-service testing against public infrastructure without authorization.
- Social engineering.
- Physical attacks.
- Vulnerabilities in third-party scanners unless caused by BataraSec integration behavior.

## Disclosure

BataraSec aims to coordinate fixes responsibly. Public disclosure should wait until a fix or mitigation is available, unless otherwise agreed.
